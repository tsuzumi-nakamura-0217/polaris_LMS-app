## 開発の進め方ガイド

### 1ステップの作業サイクル

```
① ブランチを切る（feature/○○）
② コードを書く
③ 動作確認する（ブラウザ or テスト）
④ コミットする
⑤ develop にマージする
```

### 初心者が意識すること

- **1ステップ＝1コミット** を目安にする
- 動かないコードをコミットしない（必ず動作確認してからコミット）
- 分からなくなったら `git diff` で変更箇所を確認する
- マイグレーションファイルは**必ずコミットに含める**

```python
from django.db import models
from django.conf import settings
from problems.models import Problem

class History(models.Model):
    """学習履歴"""
    problem = models.ForeignKey(
        Problem,
        on_delete=models.CASCADE,
        related_name='histories',
        verbose_name='問題'
    )
    student = models.ForeignKey(
        settings.AUTH_USER_MODEL,
        on_delete=models.CASCADE,
        related_name='histories',
        verbose_name='生徒'
    )
    level = models.IntegerField(verbose_name='理解度（1-5）')
    started_at = models.DateTimeField(verbose_name='解答開始時刻')
    finished_at = models.DateTimeField(null=True, blank=True, verbose_name='解答終了時刻')
    duration_seconds = models.IntegerField(null=True, blank=True, verbose_name='所要時間（秒）')
    is_correct = models.BooleanField(null=True, blank=True, verbose_name='正解したか')
    created_at = models.DateTimeField(auto_now_add=True)

    class Meta:
        ordering = ['-created_at']
        verbose_name = '学習履歴'
        verbose_name_plural = '学習履歴'

    def __str__(self):
        result = '○' if self.is_correct else '×'
        return f'{self.student.user_name} - {self.problem} [{result}]'
```

### ────────────────────────────────
### STEP 13: 「問題を解く」機能の実装 ★ 最重要ステップ
### ────────────────────────────────

> これが **MVP の核心** です。スケジュール → 問題表示 → 解答 → 正誤判定 → 履歴記録 → ステータス更新 を一気通貫で実装します

#### STEP 13-1: 問題表示ビューの作成
- **ファイル**: `schedules/views.py`
- **やること**: `solve_problem` ビューを作成

```python
from django.utils import timezone
from histories.models import History

@login_required
def solve_problem(request, pk):
    """スケジュールから問題を解く"""
    schedule = get_object_or_404(Schedule, pk=pk, student=request.user)
    problem = schedule.problem

    if request.method == 'GET':
        # ステータスを「進行中」に更新
        in_progress = Status.objects.get(status='進行中')
        schedule.status = in_progress
        schedule.save()

        # 解答開始時刻をセッションに保存
        request.session[f'start_time_{pk}'] = timezone.now().isoformat()

        choices = problem.choices.all() if problem.problem_type == 'choice' else None
        context = {
            'schedule': schedule,
            'problem': problem,
            'choices': choices,
        }
        return render(request, 'schedules/solve_problem.html', context)

    elif request.method == 'POST':
        # 解答を受け取る
        user_answer = request.POST.get('answer', '')
        level = int(request.POST.get('level', 3))

        # 正誤判定
        if problem.problem_type == 'text':
            is_correct = (user_answer.strip() == problem.answer.strip())
        else:
            selected_choice_id = request.POST.get('choice')
            if selected_choice_id:
                selected_choice = problem.choices.get(id=selected_choice_id)
                is_correct = selected_choice.is_correct
            else:
                is_correct = False

        # 時間計算
        start_time_str = request.session.pop(f'start_time_{pk}', None)
        started_at = timezone.datetime.fromisoformat(start_time_str) if start_time_str else timezone.now()
        finished_at = timezone.now()
        duration = int((finished_at - started_at).total_seconds())

        # 履歴を保存
        History.objects.create(
            problem=problem,
            student=request.user,
            level=level,
            started_at=started_at,
            finished_at=finished_at,
            duration_seconds=duration,
            is_correct=is_correct,
        )

        # スケジュールのステータスを「完了」に更新
        completed = Status.objects.get(status='完了')
        schedule.status = completed
        schedule.save()

        # 結果画面へ
        context = {
            'problem': problem,
            'user_answer': user_answer,
            'is_correct': is_correct,
            'level': level,
            'duration': duration,
        }
        return render(request, 'schedules/solve_result.html', context)
```

```python
from django.db.models import Count, Avg, Sum, Q

@login_required
def stats_dashboard(request):
    """統計ダッシュボード"""
    histories = History.objects.filter(student=request.user)

    # 全体の正答率
    total = histories.count()
    correct = histories.filter(is_correct=True).count()
    accuracy = (correct / total * 100) if total > 0 else 0

    # 科目別の正答率
    subject_stats = histories.values(
        'problem__category__subject__subject_name'
    ).annotate(
        total=Count('id'),
        correct=Count('id', filter=Q(is_correct=True)),
    )

    # 合計学習時間
    total_time = histories.aggregate(
        total_seconds=Sum('duration_seconds')
    )['total_seconds'] or 0

    context = {
        'total': total,
        'accuracy': accuracy,
        'subject_stats': subject_stats,
        'total_time_minutes': total_time // 60,
    }
    return render(request, 'histories/stats_dashboard.html', context)
```

- **ポイント**:

  - 科目名から `Subject` を検索（なければエラー or 自動作成）

```python
import csv
import io

@login_required
def csv_import(request):
    """CSV一括インポート"""
    if request.method == 'POST':
        form = CSVImportForm(request.POST, request.FILES)
        if form.is_valid():
            csv_file = request.FILES['csv_file']
            decoded = csv_file.read().decode('utf-8')
            reader = csv.DictReader(io.StringIO(decoded))

            created_count = 0
            errors = []

            for i, row in enumerate(reader, start=2):
                try:
                    # 科目の取得 or 作成
                    subject, _ = Subject.objects.get_or_create(
                        subject_name=row['科目名']
                    )
                    # カテゴリの取得 or 作成
                    category, _ = SubjectCategory.objects.get_or_create(
                        subject=subject,
                        title=row['カテゴリ名'],
                        defaults={'grade': int(row['学年'])}
                    )
                    # 問題の作成
                    problem = Problem.objects.create(
                        category=category,
                        problem_type=row.get('出題形式', 'text'),
                        problem=row['問題文'],
                        answer=row.get('正解', ''),
                        difficulty=int(row['難易度']) if row.get('難易度') else None,
                    )
                    # 選択式の場合は選択肢を作成
                    if problem.problem_type == 'choice':
                        labels = ['ア', 'イ', 'ウ', 'エ']
                        correct_label = row.get('正解の選択肢', '')
                        for j, label in enumerate(labels):
                            text = row.get(f'選択肢{label}', '')
                            if text:
                                Choice.objects.create(
                                    problem=problem,
                                    choice_label=label,
                                    choice_text=text,
                                    is_correct=(label == correct_label),
                                    display_order=j,
                                )
                    created_count += 1
                except Exception as e:
                    errors.append(f'{i}行目: {str(e)}')

            context = {
                'created_count': created_count,
                'errors': errors,
            }
            return render(request, 'problems/csv_import_result.html', context)
    else:
        form = CSVImportForm()

    return render(request, 'problems/csv_import.html', {'form': form})
```

## 困ったときのヒント集

### よくあるエラーと対処法

| エラー | 原因 | 対処 |
|-------|------|------|
| `No module named '○○'` | import パスが間違っている | アプリ名・モジュール名を確認 |
| `OperationalError: no such table` | マイグレーション忘れ | `makemigrations` → `migrate` を実行 |
| `TemplateDoesNotExist` | テンプレートのパスが間違っている | `templates/` からのパスを確認。`settings.py` の `TEMPLATES` 設定も確認 |
| `NoReverseMatch` | URL名が間違っている | `urls.py` の `name=` を確認。`app_name` の namespace も確認 |
| `CSRF verification failed` | フォームに `{% csrf_token %}` がない | `<form>` タグ内に追加 |
| `RelatedObjectDoesNotExist` | 外部キーの参照先が存在しない | 先にマスターデータ（科目・ステータス等）を登録する |

### 便利なコマンド

```bash
# マイグレーション関連
python manage.py makemigrations          # マイグレーションファイル作成
python manage.py migrate                  # マイグレーション実行
python manage.py showmigrations           # マイグレーション状態確認

# データ確認
python manage.py shell                    # Django Shell を起動
python manage.py dbshell                  # データベースに直接接続

# テスト
python manage.py test                     # 全テスト実行
python manage.py test problems            # 特定アプリのテストのみ

# その他
python manage.py createsuperuser          # 管理者ユーザー作成
python manage.py collectstatic            # 静的ファイル収集
python manage.py runserver                # 開発サーバー起動
```
