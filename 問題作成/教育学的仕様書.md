# Polaris LMS 教育学的仕様書

## 本書の目的

本書は、Polaris LMS の問題を作成する際の **教育学的な指針** をまとめたものです。  
AIエージェントが問題を自動生成する際に、本書を参照することで **教育的に適切で、学習効果の高い問題** を作成できるようにすることを目的としています。

本書は「問題作成依頼書（CSV仕様）」および「テンプレートCSV」と併せて使用してください。

---

## 1. 対象学習者の特性と配慮事項

### 1.1 対象学習者

本システムの主な利用者は、**児童養護施設で暮らす小学1年生〜高校3年生** の子どもたちです。

### 1.2 学習者の特性

児童養護施設で暮らす子どもたちには、以下のような学習上の特性が見られる場合があります。  
問題作成時には、これらの特性を理解し、配慮した設計を行ってください。

| 特性 | 説明 | 問題作成への影響 |
|------|------|----------------|
| **学習の空白期間** | 家庭環境の変化や転校により、特定の単元を学んでいない可能性がある | 前提知識を過度に求めない。基礎からの段階的な出題を意識する |
| **学年相当の学力差** | 実際の学年と学力が一致しない場合がある（上にも下にもずれうる） | 難易度を幅広く設定し、易しい問題から取り組めるようにする |
| **集中力の持続時間** | 心理的ストレスにより集中を保ちにくい場合がある | 1問あたりの分量を抑え、短時間で解ける問題を中心にする |
| **自己肯定感への配慮** | 失敗体験の蓄積により、学習に対する不安や苦手意識を持ちやすい | 「できた」体験を積み重ねられるよう、易しい問題を十分に用意する |
| **語彙力のばらつき** | 生活環境により、同年齢の平均的な語彙力を持たない場合がある | 問題文の表現を平易にし、専門用語にはわかりやすい言い換えを添える |
| **多様な文化的背景** | 外国にルーツを持つ子どもが含まれる場合がある | 特定の文化的知識を前提としない問題設計を心がける |

### 1.3 問題作成における基本姿勢

1. **安心して取り組める問題**：威圧的でない、穏やかな文体を使用する
2. **スモールステップ**：1つの問題で1つの概念を確認する。複合的な問題は難易度を上げて出題する
3. **生活に根ざした題材**：日常生活に関連する題材を取り入れ、学びの実感を持てるようにする
4. **トラウマインフォームド**：家族構成、経済状況、虐待を想起させる題材は避ける

## 2. 発達段階と認知特性

### 2.1 発達段階別の設計指針

| 発達段階 | 対象学年 | 認知特性 | 問題設計のポイント |
|---------|---------|---------|-----------------|
| **具体的操作期** | 小1〜小4（学年1〜4） | 具体的な事物を通じて論理的に考えられるようになる。抽象的な概念はまだ難しい | 具体的な場面設定を用いる。数の問題は実物をイメージできる文脈で出題する |
| **具体的操作期（後期）** | 小5〜小6（学年5〜6） | 抽象的思考の萌芽。分類・系列化の能力が発達する | 少しずつ抽象度を上げる。図表の読み取りや比較を取り入れる |
| **形式的操作期（前期）** | 中1〜中3（学年7〜9） | 仮説的・演繹的な思考が可能になる。抽象的な概念を扱える | 仮説を立てて検証する問題や、論理的な推論を求める問題を出題できる |
| **形式的操作期（後期）** | 高1〜高3（学年10〜12） | メタ認知が発達し、自分の思考を客観視できる | 複合的な問題、批判的思考を促す問題、論述的な問題を出題できる |

### 2.2 学年別の問題量・時間の目安

| 学年区分 | 1問あたりの問題文の長さ | 1問あたりの想定解答時間 | 1セッションの推奨問題数 |
|---------|---------------------|---------------------|---------------------|
| 小1〜小2 | 30文字以内 | 30秒〜1分 | 5〜10問 |
| 小3〜小4 | 50文字以内 | 1〜2分 | 5〜10問 |
| 小5〜小6 | 80文字以内 | 1〜3分 | 5〜10問 |
| 中1〜中3 | 150文字以内 | 2〜5分 | 5〜10問 |
| 高1〜高3 | 300文字以内 | 3〜10分 | 5〜10問 |

> ※ これは目安であり、科目や問題の性質によって変動します。

## 3. 難易度設計ガイドライン

### 3.1 難易度レベルの定義

| 難易度 | レベル名 | 教育学的位置づけ | 期待正答率 | 出題の意図 |
|:------:|---------|---------------|:---------:|-----------|
| 1 | 基礎確認 | 最低限の定着確認。学年相当の基本概念 | 90%以上 | 自信をつける。学習習慣の定着 |
| 2 | 基本定着 | 教科書の基本問題レベル | 70〜90% | 基礎力の確認と定着 |
| 3 | 標準 | 教科書の練習問題・章末問題レベル | 50〜70% | 理解の深化と応用力の基盤づくり |
| 4 | 応用 | 既習内容を組み合わせて解く問題 | 30〜50% | 思考力・応用力の育成 |
| 5 | 発展 | 教科書の枠を超えた挑戦的な問題 | 10〜30% | 発展的学習。学ぶ意欲の高い子への対応 |

### 3.2 難易度バランスの推奨比率

1つのカテゴリ（単元）につき、以下のバランスで問題を作成してください。

| 難易度 | 推奨比率 | 理由 |
|:------:|:-------:|------|
| 1（基礎確認） | 25% | 「できた」体験を通じて学習意欲を維持する |
| 2（基本定着） | 30% | 基礎の確実な定着を図る |
| 3（標準） | 25% | 理解を深め、応用への橋渡しをする |
| 4（応用） | 15% | 考える力を養い、達成感を得る |
| 5（発展） | 5% | 学力の高い子どもに挑戦の機会を提供する |

> **重要**: 児童養護施設の子どもたちは学習の空白期間を持つことが多いため、通常の教材よりも **難易度1〜2の問題を厚めに** 用意してください。「簡単すぎる」と感じる問題でも、自信を取り戻すために重要です。

### 3.3 スモールステップ設計

1つの単元内で、以下のように段階的に難易度を上げていく設計を推奨します。

```
ステップ1（難易度1）: 概念の確認
   例）「3 + 2 はいくつですか？」→ 基本的な計算ができるか確認

ステップ2（難易度2）: パターンの反復
   例）「7 + 5 はいくつですか？」→ 繰り上がりを含む計算

ステップ3（難易度3）: 文章題への展開
   例）「りんごが3こ、みかんが5こあります。あわせていくつですか？」

ステップ4（難易度4）: 条件の複合
   例）「りんごが何こかあります。5こ食べたら2こ残りました。はじめは何こありましたか？」

ステップ5（難易度5）: 発展・思考力
   例）「□ + △ = 10 となる□と△の組み合わせをすべて書きなさい。」
```

## 4. 出題形式の使い分け

> **方針**：本システムでは **選択式（choice）を中心** に出題します。選択式は解答の心理的ハードルが低く、正誤判定も正確なため、児童養護施設の学習者に適しています。記述式は、計算問題の答えや漢字の読みなど **正解が一意に定まる場合のみ** 使用してください。  
> **目安**：1カテゴリ100問のうち、**選択式70%以上・記述式30%以下** を推奨します。

### 4.1 記述式（text）を使うべき場面

| 場面 | 理由 | 例 |
|------|------|-----|
| 計算問題 | 正解が一意に定まるため | 「3 + 5 はいくつですか？」 |
| 漢字の読み | ひらがなで一意に表記できるため | 「『図書館』の読みをひらがなで書きなさい。」 |
| 英単語のスペル | 正確なスペルを書く練習が必要な場合 | 「『りんご』を英語で書きなさい。」 |
| 短い用語の解答 | 表記ゆれが発生しにくい用語 | 「日本で一番長い川の名前を答えなさい。」 |

### 4.2 選択式（choice）を使うべき場面

| 場面 | 理由 | 例 |
|------|------|-----|
| 正解の表記にゆれが生じる問題 | 完全一致判定のため誤判定を防ぐ | 「次のうち正しいものを選びなさい。」 |
| 概念の理解を確認する問題 | 文字を書くことが目的ではない場合 | 「光合成を行う部分はどれですか？」 |
| 低学年・英語初学者向け | テキスト入力の負担を軽減する | 「『犬』を英語で表すとどれですか？」 |
| 歴史の人物・地名の問題 | 漢字の表記ゆれを避けるため | 「関ヶ原の戦いの勝者は誰ですか？」 |
| 定義・意味を問う問題 | 微妙なニュアンスの違いを判別させる | 「『猿も木から落ちる』の意味は？」 |
| 間違えやすいポイントを学ぶ問題 | 誤答の選択肢自体に学習効果がある | 「次のうち、三角形の面積の公式はどれ？」 |

### 4.3 使い分けの判断フロー

```
正解が1つに定まるか？
   │
   ├─ はい → テキスト入力で正確に記述できるか？
   │           │
   │           ├─ はい → 記述式（text）
   │           │         例：計算の答え、ひらがなの読み
   │           │
   │           └─ いいえ → 選択式（choice）
   │                       例：漢字の表記ゆれ、英語の大文字小文字
   │
   └─ いいえ → 選択式（choice）
               例：意味の理解、概念の判別
```

## 5. 問題文の書き方

### 5.1 文体・表現のルール

| 学年区分 | 文体 | 注意点 |
|---------|------|-------|
| 小1〜小2 | ひらがな中心。やさしい口調 | 漢字を使う場合は学年配当漢字のみ。一文は20文字以内を目安に |
| 小3〜小4 | 常用漢字（学年配当内）＋ひらがな | 難しい言い回しは避ける。一文は30文字以内を目安に |
| 小5〜小6 | 「〜しなさい」「〜を答えなさい」 | 指示語を明確にする。条件は箇条書きにする |
| 中1〜高3 | 「〜を求めよ」「〜を述べよ」も可 | 専門用語は教科書の表現に準拠する |

### 5.2 指示文の定型パターン

問題の末尾に付ける指示文は、以下のパターンを推奨します。

| 出題意図 | 記述式の指示文 | 選択式の指示文 |
|---------|-------------|-------------|
| 答えを求める | 「答えなさい。」「求めなさい。」 | 「正しいものを選びなさい。」 |
| 読みを答える | 「ひらがなで書きなさい。」 | 「正しい読みを選びなさい。」 |
| 計算結果を答える | 「計算しなさい。」 | 「答えとして正しいものを選びなさい。」 |
| 用語を答える | 「何といいますか。」 | 「正しい用語を選びなさい。」 |
| 英語で答える | 「英語で書きなさい。」 | 「正しい英語を選びなさい。」 |

### 5.3 避けるべき表現

| 避けるべき表現 | 理由 | 代替表現 |
|-------------|------|---------|
| 「お父さんとお母さんが〜」 | 家庭環境への配慮 | 「太郎さんと花子さんが〜」 |
| 「おうちで〜」 | 帰る家がない子への配慮 | 具体的な場所名（「教室で」「公園で」等） |
| 「家族旅行で〜」 | 家族構成への配慮 | 「校外学習で〜」「遠足で〜」 |
| 「お小遣いで〜」 | 経済状況への配慮 | 「〇〇円持っています。」 |
| 「当たり前のことですが〜」 | 自己肯定感への配慮 | 使用しない |
| 「簡単な問題です」 | 解けなかった場合の心理的負担 | 使用しない |
| 「〜ではないものを選べ」 | 否定形の出題は混乱を招く | 「〜にあてはまるものを選びなさい」 |

### 5.4 登場人物名のルール

問題文に人物が登場する場合は、以下のガイドラインに従ってください。

- **一般的な日本の名前** を使用する（太郎、花子、たけし、さくら 等）
- **多様な名前** を含めることを意識する（海外にルーツを持つ名前も適宜使用）
- **性別による役割の固定化** を避ける（「太郎くんは計算が得意で、花子さんは国語が得意」のような表現）
- 人物名は **1問につき2名まで** にする（名前が多いと混乱する）

## 6. 選択肢の設計原則

### 6.1 良い選択肢の条件

| 原則 | 説明 |
|------|------|
| **もっともらしさ** | 不正解の選択肢（ディストラクター）も、一見正しそうに見えるものにする |
| **学習効果** | 不正解の選択肢は、よくある間違いや誤解を反映したものにする |
| **均一な長さ** | 選択肢の文字数をなるべく揃える。正解だけ長い／短いとヒントになる |
| **独立性** | 各選択肢は互いに排他的であること。重複する意味の選択肢を入れない |
| **文法的整合性** | すべての選択肢が問題文の続きとして文法的に正しくなるようにする |

### 6.2 ディストラクター（誤答選択肢）の設計

効果的なディストラクターは、以下のような **よくある間違い** に基づいて作成します。

| 間違いの種類 | 例 | 科目 |
|------------|-----|------|
| 計算ミス | 3 × 4 = 7（たし算と混同） | 算数・数学 |
| 類似語の混同 | 「意外」と「以外」 | 国語 |
| スペルミス | recieve（正：receive） | 英語 |
| 概念の混同 | 光合成を「根」で行うと誤答 | 理科 |
| 時代の混同 | 織田信長が江戸幕府を開いたと誤答 | 社会 |
| 公式の誤記憶 | 三角形の面積 = 底辺 × 高さ（÷2を忘れる） | 算数・数学 |
| 単位の間違い | 1km = 100m（正：1000m） | 算数 |

### 6.3 選択肢の並び順

- **数値の選択肢**：小さい順に並べる（ア=7, イ=10, ウ=12, エ=15）
- **時系列の選択肢**：古い順に並べる
- **文字列の選択肢**：五十音順またはアルファベット順を基本とする
- **正解の位置**：ア〜エにまんべんなく分散させる（偏りが出ないように）

## 7. 解説の書き方

すべての問題に **解説** を付けてください。解説は、正解・不正解にかかわらず生徒が学びを深めるための重要な要素です。

### 7.1 解説に含めるべき内容

| 要素 | 説明 | 例 |
|------|------|-----|
| **正解の理由** | なぜその答えが正しいのかを説明する | 「3 + 5 = 8 です。3と5をあわせると8になります。」 |
| **考え方・解き方** | 解答にたどり着くプロセスを示す | 「両辺から3を引くと2x=8。両辺を2で割るとx=4です。」 |
| **よくある間違い** | 選択式の場合、なぜ他の選択肢が不正解なのかを補足する | 「根は水や養分を吸収する部分で、光合成は行いません。」 |
| **関連知識** | 問題に関連する補足情報を簡潔に添える | 「富士山は標高3776mで静岡県と山梨県の境にあります。」 |

### 7.2 解説の書き方ルール

1. **わかりやすい言葉** で書く。対象学年の語彙レベルに合わせる
2. **簡潔に** まとめる。1問あたり50〜150文字程度を目安にする
3. **肯定的な表現** を使う。「〜が間違いです」より「〜が正しいです。なぜなら〜」の形式で
4. 低学年向けはひらがな中心で記述する（問題文と同じ文体ルールに従う）
5. 公式や法則を使った問題では、**公式の意味** も併せて説明する
6. 解説内でも **避けるべき表現**（6.3節参照）に注意する

### 7.3 科目別の解説ポイント

| 科目 | 解説のポイント |
|------|-------------|
| 算数・数学 | 計算の途中経過を示す。図形問題は条件の読み取り方を説明する |
| 国語 | 漢字の成り立ちや語源を添える。文法は具体例を使って説明する |
| 英語 | 単語の覚え方のヒントを添える。文法は日本語との違いを説明する |
| 理科 | 身近な現象と結びつけて説明する。実験の原理をわかりやすく述べる |
| 社会 | 歴史的背景や地理的な理由を添える。暗記でなく「なぜ」を説明する |

## 8. 復習効果を高める問題設計

### 8.1 スパイラル学習の原則

本システムでは、間違えた問題や理解度の低い問題を自動的に復習提案する機能があります。  
この機能を最大限に活かすため、以下の設計を意識してください。

1. **同じ概念を異なる角度から出題** する複数の問題を用意する
   - 例：「3 + 5 = ?」と「□ + 5 = 8, □にあてはまる数」は同じたし算だが異なる思考力を使う
2. **同じ問題の難易度違いバージョン** を作成する
   - 例：難易度1「5 × 3 = ?」→ 難易度3「1箱に5こ入ったチョコが3箱あります。全部で何こ？」
3. **誤答パターンに対応する補強問題** を用意する
   - 例：「12 - 5 = ?」を間違えた生徒向けに「12 - 3 = ?」「11 - 5 = ?」などの類題を多めに

### 8.2 1カテゴリあたりの推奨問題数

| 難易度 | 推奨比率 | 推奨問題数（100問の場合） | 用途 |
|:------:|:-------:|:----------------------:|------|
| 1 | 25% | 25問 | 基礎の定着確認用。反復学習向き |
| 2 | 30% | 30問 | 基本力を固めるための練習用 |
| 3 | 25% | 25問 | 標準レベルの理解度確認用 |
| 4 | 15% | 15問 | 応用力の確認用 |
| 5 | 5% | 5問 | 発展学習・チャレンジ用 |
| **合計** | **100%** | **100問** | 1カテゴリあたり |

> 1カテゴリあたり **100問** を作成してください。システムはこの中からランダムに10問を出題します。問題のバリエーションを豊富にすることで、繰り返し学習しても新鮮な問題に取り組めるようにします。

## 9. 品質チェックリスト

問題を作成した後、以下の観点で品質を確認してください。

### 教育的観点

- [ ] 学年の発達段階に適切な内容か
- [ ] 学習指導要領の範囲内か
- [ ] 難易度の設定は適切か（期待正答率と合っているか）
- [ ] スモールステップになっているか（唐突に難しくなっていないか）
- [ ] 問題文は簡潔で理解しやすいか

### 配慮事項

- [ ] 家庭環境に関する不適切な表現がないか
- [ ] 経済状況を前提とした表現がないか
- [ ] 特定の文化・国を否定的に扱っていないか
- [ ] 性別による役割の固定化がないか
- [ ] 威圧的・否定的な表現がないか
- [ ] 暴力・虐待を想起させる表現がないか

### 技術的観点

- [ ] 記述式の正解は一意に定まるか（表記ゆれの可能性がないか）
- [ ] 選択式のディストラクターは適切か（もっともらしいか）
- [ ] 問題文が500文字以内に収まっているか
- [ ] 正解・選択肢が200文字以内に収まっているか
- [ ] すべての問題に解説が付いているか
- [ ] 解説は対象学年の語彙レベルに合っているか
- [ ] 解説に避けるべき表現が含まれていないか
- [ ] 1カテゴリあたり100問作成されているか
- [ ] 選択式の比率が70%以上か

## 10. 付録：学年配当漢字の目安

問題文で漢字を使用する際の参考情報です。問題文に使用する漢字は、原則として **対象学年までに習う漢字** にとどめてください。

| 学年 | 配当漢字数（累計） | 特記事項 |
|:----:|:----------------:|---------|
| 小1 | 80字 | 基本的な漢字。問題文はひらがな中心 |
| 小2 | 240字（+160字） | 簡単な熟語が使えるようになる |
| 小3 | 440字（+200字） | 問題文に漢字を使い始める |
| 小4 | 642字（+202字） | 一般的な文章表現が可能に |
| 小5 | 835字（+193字） | 抽象的な概念を表す漢字が増える |
| 小6 | 1,026字（+191字） | 小学校で習う漢字はほぼ網羅 |
| 中学 | 常用漢字（2,136字） | 中学校で残りの常用漢字を学習 |

> 対象学年より上の漢字を使用する場合は、ふりがなを括弧書きで添えてください。  
> 例：「光合成（こうごうせい）」
