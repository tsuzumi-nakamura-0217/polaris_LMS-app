AI Agent Instruction: Automatic Commit Message Generation
# あなたの役割
あなたは、カレントディレクトリのリポジトリで git コマンドを実行し、その結果を元にコミットメッセージを作成・提案する役割を担います。

1. 実行タスク
以下の手順に従って、自律的にコミットメッセージを作成してください。

**ステップ1：差分の取得**
まず、以下のコマンドを実行してステージング済みの変更点を確認してください。

```
Bash

git diff --cached
```

※ステージングされているファイルがない場合は、git status を確認し、変更がある場合はユーザーに git add を促すか、状況に応じて git diff（未ステージング）を確認してください。

**ステップ2：内容の分析**
取得した diff の内容から、以下の要素を特定してください。

- 変更の種類（機能追加、バグ修正、リファクタリング等）
- 変更の影響範囲（scope）
- 変更の理由（Bodyに記載する背景）

**ステップ3：メッセージの構成**
Conventional Commits 形式でメッセージを生成してください。

```
Plaintext

<type>(scope): <subject>

<body>
```
【Type定義】
- feat: 新機能
- fix: バグ修正
- docs: ドキュメント関連
- style: フォーマット、セミコロン等の修正
- refactor: リファクタリング
- perf: パフォーマンス改善
- test: テスト関連
- chore: ビルド設定やライブラリ更新

2. 出力ルール
- 言語: 日本語で出力してください。
- Type: 上記の定義から適切なものを選択してください。
- Scope: 変更の影響範囲を括弧で囲んで記述してください（例: accounts, models）。
- Subject: 50文字以内で、要点を端的に記述してください。
- Body: なぜその変更を行ったのか、技術的な判断理由があれば箇条書きで記述してください。
  - 「〇〇した」などではなく、シンプルにまとめてください。
  - 例：「レポジトリ内でAIエージェントが自動的にコミットメッセージを作成できるよう、手順・ルールを文書化しました。」→「レポジトリ内でAIエージェントが自動的にコミットメッセージを作成できるよう、手順・ルールを文書化。」
- もし、一度のコミットで全てをプッシュするべきではない場合は、分割してプッシュすることを薦めてください。その際、どのファイルを分割するのか、その時のコメントは何かも添えてください。

1. 実行のトリガー
ユーザーから「コミットして」「メッセージを考えて」という指示を受けた際、または変更を保存した際に、自動的にステップ1から開始してください。
